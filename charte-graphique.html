06 — Règles d'usage

Do & Don'ts

DO — À faire
Laisser une zone de protection d'au moins un demi-hexagone autour du logo
Utiliser la palette limitée (3 couleurs vives maximum simultanément)
Vérifier le contraste WCAG AA minimum avant publication
Privilégier la lisibilité : texte blanc sur Bleu Profond, texte sombre sur Neutre Clair
Utiliser l'Orange uniquement pour les CTA et signaux d'action
Ajouter un alt text descriptif à chaque image publiée
DON'T — À éviter
Déformer ou étirer le logo (conserver les proportions à tout moment)
Utiliser plus de 3 couleurs vives simultanément sur un même visuel
Superposer du texte sur des images sans contraste suffisant
Placer le logo sur un fond de couleur similaire (utiliser version contrastée)
Utiliser des polices hors charte (Arial, Helvetica générique) pour le branding
Utiliser le logo compact en dessous de 40px (rendu illisible)
Checklist — Publication LinkedIn
Logo compact visible en coin bas-droit
Titre sous 140 caractères
Image dimensionnée 1200×627 px (post) ou 1080×1080 px (carousel)
CTA visuel en Orange si action attendue
Contraste texte/fond vérifié (WCAG AA)
Alt text rédigé pour l'accessibilité
Checklist — GitHub Pages
Hero banner 1600×600 px avec tagline
README projet : 1 phrase d'accroche + 3 bullets
Captures de code en Space Mono / JetBrains Mono
Mini-cards projets 360×220 px avec tags Data/Sec/MLOps
Contraste WCAG AA vérifié sur tous les composants
Tokens CSS implémentés (--brand-primary, --accent-teal, --alert)
